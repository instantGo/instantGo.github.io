Componente Copy

Botón inteligente para copiar texto al portapapeles con feedback visual, soporte multilenguaje y total compatibilidad con Turbo.

Dependencias

  • Stimulus (v3.2+): El controlador está construido sobre Stimulus. Asegúrate de tenerlo instalado y configurado en tu proyecto.
  • Bootstrap CSS: El componente utiliza clases de Bootstrap para los estilos del botón y el placeholder. Es necesario tener Bootstrap incluido en tu proyecto.
  • Bootstrap Icons: Se usan para los iconos por defecto. Si prefieres otros iconos, puedes personalizarlos mediante los atributos correspondientes.
  • Turbo (opcional): El componente es totalmente compatible con Turbo Drive, pero no requiere que Turbo esté instalado para funcionar. Si está presente, el controlador se integrará automáticamente.
  • ES Module Shims (recomendado para navegadores antiguos): Si necesitas soporte para navegadores que no implementan import maps (por ejemplo, versiones antiguas de Safari, Edge o Chrome), incluye el shim correspondiente.

Instalación rápida

Para usar el componente directamente en una página HTML, sigue estos pasos:

  1. Incluye las dependencias en el <head>
    Añade los siguientes enlaces para Bootstrap CSS, Bootstrap Icons y ES Module Shims (opcional pero recomendado para navegadores antiguos): 
    <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/css/bootstrap.min.css">
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/bootstrap-icons@1.13.1/font/bootstrap-icons.min.css">
<script async src="https://cdn.jsdelivr.net/npm/es-module-shims@2.8.0/dist/es-module-shims.js"></script>
  2. Configura un import map
    Define las rutas de los módulos que vas a usar (Stimulus y el controlador). Stimulus desde un CDN y el controlador desde local: 
    <script type="importmap">
    {
      "imports": {
        "@hotwired/stimulus": "https://cdn.jsdelivr.net/npm/@hotwired/stimulus@3.2.2/+esm",
        "instant_copy_controller": "/ruta/a/tu/archivo/instant_copy_controller.js"  // Ajusta la ruta según donde hayas colocado el controlador
      }
    }
</script>
  3. Registra el controlador en la página
    Agrega un script tipo módulo que inicie Stimulus y registre el controlador: 
    <script type="module">
      import { Application } from '@hotwired/stimulus'
      import InstantCopyController from 'instant_copy_controller'
    
      const app = Application.start()
      app.register('instant-copy', InstantCopyController)
    </script>
  4. ¡Listo!
    Ya puedes usar el componente en cualquier parte de tu página con la estructura HTML adecuada.

Asegúrate de que la ruta al archivo instant_copy_controller.js en el import map sea correcta.

Estructura HTML mínima del componente

Si ya tienes las dependencias y el controlador registrado, este es el HTML mínimo que necesitas para que el componente funcione:

<div data-controller="instant-copy" data-instant-copy-text-value="Texto a copiar">
    <span data-instant-copy-target="placeholder" class="btn btn-sm placeholder" style="width: 100px;"></span>
    <button data-action="click->instant-copy#copy"
            data-instant-copy-target="button"
            class="btn btn-sm btn-info d-flex flex-wrap gap-2 d-none">
        <i aria-hidden="true" class="bi bi-clipboard"></i>
        <span data-instant-copy-target="buttonText"></span>
    </button>
</div>

Este bloque puede ir en cualquier lugar de tu página. Ajusta data-instant-copy-text-value o data-instant-copy-target-value según necesites.

Atributos configurables

Estos atributos se colocan en el mismo elemento que lleva data-controller="instant-copy". Todos deben terminar en -value, por ejemplo: data-instant-copy-lang-value="en".

data-instant-copy-text-value

Texto literal que se copiará al hacer clic. Si se usa, tiene prioridad sobre target a menos que este último esté presente como parámetro en la URL de un turbo-frame que esté cargando al componente.

Tipo: String | Default: —

data-instant-copy-target-value

Selector CSS (como #id, .clase, [atributo]) del elemento del que se extraerá el texto. Funciona con elementos normales (div, span) y también con input, textarea y select (tomando su valor).

Tipo: String (selector) | Default: —

data-instant-copy-lang-value

Idioma de los textos del botón y feedback. Puede ser sobrescrito por el parámetro lang en la URL de la página o de un turbo-frame.

Tipo: "es" o "en" | Default: "es"

data-instant-copy-translations-value

Objeto JSON que permite personalizar las traducciones. Puedes sobrescribir los textos por defecto para un idioma existente (es/en) o añadir traducciones para un nuevo idioma. El objeto debe tener la misma estructura que las traducciones por defecto.

Tipo: JSON string | Default: "{}"

Ejemplo para sobrescribir español y añadir francés:
'{"es":{"button":"Copiar","success":"¡Listo!"},"fr":{"button":"Copier","success":"Copié !","error":"Erreur"}}'
data-instant-copy-success-class-value

Clases CSS que se aplican al botón durante el feedback de éxito. Puedes usar cualquier clase de Bootstrap o personalizadas.

Tipo: String | Default: "text-success bg-opacity-10"

data-instant-copy-error-class-value

Clases CSS que se aplican al botón durante el feedback de error.

Tipo: String | Default: "text-danger bg-opacity-10"

data-instant-copy-icon-success-value

Clase del icono de Bootstrap Icons que se muestra en el feedback de éxito. Puedes usar cualquier clase de Bootstrap Icons.

Tipo: String | Default: "bi-check-circle-fill"

data-instant-copy-icon-error-value

Clase del icono que se muestra en el feedback de error.

Tipo: String | Default: "bi-exclamation-triangle-fill"

data-instant-copy-remove-classes-value

Lista de clases CSS exactas (separadas por espacios) que se eliminarán temporalmente del botón durante el feedback. Útil para evitar interferencias de clases de color/fondo/borde personalizadas.

Tipo: String | Default: "" (vacío)

Ejemplo: "text-primary-emphasis bg-primary-subtle border-primary-subtle"

Prioridades

El componente sigue un orden jerárquico para determinar tanto el texto a copiar como el idioma.

Prioridad del texto a copiar

  1. Parámetros de URL de un <turbo-frame>
    Si el componente está dentro de un frame con src y la URL contiene ?target=..., este tiene máxima prioridad.
  2. Atributos HTML del componente
    data-instant-copy-text-value o data-instant-copy-target-value definidos en el elemento.
  3. Si no se especifica ninguna fuente
    Se muestra feedback de error.

Si coexisten text y target como atributos, se usa text y se ignora target.

Prioridad del idioma

  1. Parámetro lang en la URL de un <turbo-frame>
    Si el componente está dentro de un frame con src y la URL contiene ?lang=..., este valor tiene prioridad.
  2. Parámetro lang en la URL de la página
    Se busca en la URL principal.
  3. Atributo data-instant-copy-lang-value
    Valor definido en el elemento (o "es" por defecto).

Esto permite cambiar el idioma globalmente mediante un parámetro en la URL.

Ejemplo combinado: <turbo-frame src="/copy?target=#idtext&lang=en">...</turbo-frame> usará el texto del elemento identificado con el idtext e idioma inglés, independientemente de los atributos del componente.

Targets del controlador

Los targets son elementos del DOM que el controlador utiliza para interactuar. Cada uno se identifica con el atributo data-instant-copy-target="nombre". A continuación se detalla cada uno:

data-instant-copy-target="placeholder"

Propósito: Evitar un salto visual mientras Stimulus inicializa el controlador. Se muestra inicialmente y el controlador lo oculta (añadiendo d-none) cuando está listo, mostrando el botón real.

Recomendaciones: Debe tener un tamaño similar al botón para evitar reajustes. Usa clases de placeholder de Bootstrap (como btn placeholder col-*) o fijar el tamaño (style ="width: 100px;") y el contenedor padre puede llevar placeholder-glow para un efecto de carga.

Ejemplo: <span data-instant-copy-target="placeholder" class="btn btn-sm bg-info-subtle border border-info-subtle rounded-3 placeholder col-3"></span>

data-instant-copy-target="button"

Propósito: Botón que ejecuta la acción de copiado. Es el elemento principal que dispara copy().

Requisitos: Debe tener data-action="click->instant-copy#copy" y la clase inicial d-none (oculto). El controlador lo muestra automáticamente al conectarse.

Contenido opcional en el elemento contenedor (button) de este target:

  • Un icono (ej. <i class="bi bi-clipboard" aria-hidden="true"></i>). Si está presente, el controlador cambiará su clase durante el feedback (a iconSuccess o iconError). Si no hay icono, el feedback visual se basará en cambios solo de texto si el target buttonText está presente y clases ( successClass o errorClass) que afectan al propio elemento (button).
  • El target buttonText (necesario si se desea que el texto del botón se visualice y se actualice durante el feedback). Si no esta presente el feedback se basará en el cambio del icon mediante (iconSuccess o iconError) si está presente y las clasess ( successClass o successClass) que afectan al propio elemento (button)
  • El target live (recomendado para accesibilidad).

Ejemplo: <button data-action="click->instant-copy#copy" data-instant-copy-target="button" class="btn btn-sm text-primary-emphasis bg-primary-subtle border border-primary-subtle rounded-3 d-none"> ... </button>

data-instant-copy-target="buttonText"

Propósito: Elemento (normalmente un <span>) dentro del botón que muestra el texto dinámico.

Comportamiento: El controlador actualiza su contenido según el idioma y el estado: texto normal (clave button), éxito (success) o error (error). Puede estar vacío inicialmente.

Importante: Este target es necesario para que el texto del botón se visuaalize y cambie durante el feedback. Si no se incluye, el texto no se visualizará, pero el resto del feedback (icono y clases) seguirá funcionando.

Ejemplo: <span data-instant-copy-target="buttonText"></span>

data-instant-copy-target="live"

Propósito: Región ARIA live para proporcionar feedback accesible a usuarios de lectores de pantalla. Anuncia el resultado de la copia (éxito o error) sin necesidad de que el foco esté en el botón.

Uso recomendado: Incluir un <span> con las clases visually-hidden y los atributos aria-live="polite" y aria-atomic="true". El controlador actualizará su contenido con los mensajes livesuccess o liveerror (según idioma) y lo limpiará al restaurar el estado.

Nota: No es obligatorio para el funcionamiento básico, pero se recomienda encarecidamente para cumplir con estándares de accesibilidad.

Ejemplo: <span data-instant-copy-target="live" class="visually-hidden" aria-live="polite" aria-atomic="true"></span>

Importante: El target data-instant-copy-target="buttonText" es necesario para que el controlador pueda visualizar y cambiar el texto durante el feedback. Si no se incluye, el componente funcionará pero el texto del botón no se visualizará.
Accesibilidad: Se recomienda incluir el target data-instant-copy-target="live" con los atributos ARIA adecuados para que los usuarios de lectores de pantalla reciban notificación del resultado de la copia.

Acción

El controlador expone una única acción que debe enlazarse al evento click del botón.

copy

Ejecuta la copia del texto al portapapeles y gestiona el feedback visual correspondiente.

Forma de enlazarla:

<button data-action="click->instant-copy#copy" ...>
Qué hace internamente:
  1. Obtiene el texto a copiar según la fuente configurada (texto directo, target como atributo o parámetro en la URL del frame) teniendo en cuenta la prioridad de estos.
  2. Verifica que la API navigator.clipboard esté disponible.
  3. Ejecuta writeText() para copiar el texto.
  4. Muestra feedback de éxito o error durante 2 segundos.
  5. Restaura el botón a su estado original.
Nota: No es necesario llamar a esta acción manualmente desde otro código. Solo se usa mediante el atributo data-action en el botón.
Importante: Asegúrate de que el botón tenga el atributo data-action="click->instant-copy#copy". Si no se incluye, al hacer clic no ocurrirá nada.

Uso con Turbo Frames: Para utilizar esta funcionalidad, Turbo debe estar instalado en tu proyecto. El componente puede cargarse dentro de un <turbo-frame> que apunte a una URL que contiene el HTML del componente. Los parámetros incluidos en la URL del frame (como ?target=... , ?lang=...) tienen prioridad sobre los atributos definidos directamente en el HTML del componente. Esto permite pasar valores dinámicos sin modificar la página que aloja el componente.
Página principal (ej. /pagina):
<turbo-frame id="componente-instant-copy" src="/componentes/instant-copy?target=#idtext&lang=en">
    <!-- Mientras carga, puede mostrar un placeholder -->
    Cargando...
</turbo-frame>
Página del componente (ej. /componentes/copy):
<turbo-frame id="componente-instant-copy">
     <div data-controller="instant-copy"
        data-instant-copy-text-value="Texto por defecto"
        data-instant-copy-lang-value="es"
        class="placeholder-glow">
        <span data-instant-copy-target="placeholder" class="btn btn-sm placeholder" style="width: 100px;"></span>
        <button data-action="click->instant-copy#copy"
               data-instant-copy-target="button"
               class="btn btn-sm btn-info d-flex flex-wrap gap-2 d-none">
               <i aria-hidden="true" class="bi bi-clipboard"></i>
               <span data-instant-copy-target="buttonText"></span>
        </button>
    </div>
</turbo-frame>

Resultado: El componente se cargará usando el texto del elemento identificado con #idtext y el idioma inglés, ignorando los valores por defecto definidos en su HTML (data-instant-copy-text-value="Texto por defecto" y data-instant-copy-lang-value="es").

Necesitas tener Turbo instalado como dependencia para usar esta característica excepto para el parámetro lang que puede ser pasado como parámetro en la url global de la página teniendo el mismo efecto y prioridad con respecto a los atributos en el HTML del componente

Comportamiento general

El componente sigue un flujo predecible en todas sus interacciones. Estos son los aspectos clave de su funcionamiento:

  • Al hacer clic: Se obtiene el texto a copiar siguiendo la prioridad definida (parámetro de frame → atributos HTML). Si no hay texto, se muestra feedback de error.
  • Éxito: Si la copia es exitosa, el botón muestra durante 2 segundos el icono y mensaje de éxito configurados (data-instant-copy-icon-success-value y traducción success), y se aplican las clases de data-instant-copy-success-class-value.
  • Error: Si falla (API no disponible, elemento no encontrado, texto vacío), se muestra feedback de error con su icono, mensaje y clases correspondientes.
  • Restauración: Tras 2 segundos, el botón recupera su estado original: texto del botón, icono y clases iniciales.
  • Clics durante el feedback: Si se pulsa el botón mientras está mostrando un feedback, el feedback anterior se cancela y se inicia uno nuevo (primero se restaura el estado original y luego se aplica el nuevo feedback).
  • Desconexión del controlador: Cuando el elemento es eliminado del DOM (por ejemplo, al navegar con Turbo), el controlador restaura el estado original antes de desconectarse, dejando el DOM limpio.
  • Con Turbo: Además de la restauración en desconexión, escucha el evento turbo:before-cache para asegurar que la página se cachea con el botón en su estado original, evitando que aparezca con feedback al volver atrás.
  • Sin texto configurado: Si no se define ninguna fuente de texto (ni atributo ni parámetro de frame), al hacer clic se mostrará directamente el feedback de error.
  • Eliminación de clases con removeClasses: Durante el feedback, se eliminan temporalmente las clases listadas en data-instant-copy-remove-classes-value para evitar interferencias visuales. Luego se restauran junto con el estado original.
  • Internacionalización: El componente incluye traducciones por defecto en español e inglés. Puedes extenderlo a otros idiomas mediante data-instant-copy-translations-value. Si no se encuentra el idioma solicitado, se usa español como fallback.
  • Placeholder: El elemento placeholder se oculta automáticamente al conectar el controlador. Si el controlador falla por algún motivo, el placeholder permanecerá visible, lo que puede servir como indicador de error.
  • Múltiples instancias: Puedes usar tantos componentes Copy en una misma página como necesites. Cada uno funciona de forma independiente, con su propia configuración y estado.
Tiempo de feedback: Los 2 segundos son fijos por diseño. Si necesitas cambiarlo, puedes modificar el valor en el controlador (línea del setTimeout).
